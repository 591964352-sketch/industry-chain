# 单源核实 Skill

## 触发条件

任何 `*.manual.js` 文件（PCB/HBM/光模块/半导体设备 等所有产业链）的任何字段里出现 `⚠️单源待核` 标注时启用。

不适用范围（明确豁免）：
- `pcb.auto.js` / `<chain>.auto.js` 的 `flag` / `source.flag` 字段（程序生成的 flag，非手动层 ⚠️）
- `index.html` 渲染层的 UI 提示（如 `⚠️ 高于 PE 上限` 等硬编码提示，非手动层 ⚠️）
- `changelog` 数组里的历史变更标注（变更日志，非当前数据）

## 执行步骤

### 第一步：生成豆包提示词

对每条 `⚠️单源待核` 生成以下格式的提示词给用户复制到豆包 / Claude / 其他 AI 平台：

---

```
你是严格的投研数据核实助手。

【硬约束·必须遵守·违反则本次核实无效】
1. 只引用可明确溯源的信息（公司公告/券商研报/行业媒体）
2. 严禁推测、编造、模糊表述（"可能""据悉""有消息称"等一律无效）
3. 找不到第二来源必须直接说"未找到第二来源"，不得绕过
4. 每条信息必须同时注明：来源名称 + 发布日期 + 原文关键句，缺一无效
5. 豆包自身的训练数据不算独立来源，必须是可查阅的外部文件

【核实任务】
标的：{stock_name}（{stock_code}）
待核事实：{fact}
原始来源：{original_source}（{original_date}）
目标：在原始来源之外，找到至少1个独立来源支撑上述事实

【搜索范围（按优先级）】
1. 巨潮资讯 cninfo.com.cn · {stock_code} 近6个月公告
2. 券商研报 · 覆盖该标的或所在行业的近期报告
3. 行业媒体 · 电子工程专辑/华强电子/PCB行业网
4. 可比公司 · 同赛道竞争对手公告/年报中是否提及同类进展

【必须按此格式输出】
结论：找到第二来源 / 未找到第二来源
来源列表：
  - 来源名称：
  - 发布日期：
  - 原文关键句：
建议：可以去掉⚠️单源待核 / 保留⚠️单源待核
```

---

### 第二步：等待用户把豆包结果贴回

用户在 Claude Code 中执行（或独立窗口粘贴豆包回复），把豆包的回复原文贴回当前会话。

### 第三步：判断规则（硬约束）

| 豆包返回 | 判断 | 处理 |
|---|---|---|
| 「找到第二来源」且来源信息**完整**（名称 + 日期 + 原文关键句 三项齐全） | ✅ 有效 | 可以去掉 ⚠️ |
| 「找到第二来源」但来源信息**不完整**（缺名称 / 日期 / 原文任一项）| ❌ 无效 | 保留 ⚠️ |
| 「未找到第二来源」 | ✅ 明确 | 保留 ⚠️ |
| 回复含模糊表述（可能 / 据悉 / 听说 / 市场传言 / 有消息指出 / 大概）| ❌ 无效 | 保留 ⚠️ |
| 来源是「豆包训练数据」/ 「通用知识」/ 「行业普遍认知」| ❌ 无效 | 保留 ⚠️ |
| 来源不是公司公告 / 券商研报 / 行业媒体（仅个人公众号/雪球散户帖/股吧）| ❌ 无效（按 CLAUDE.md §6.4 media 单源规则）| 保留 ⚠️ |

### 第四步：修改 manual.js

仅当「可以去掉⚠️」时执行修改：

**改前**：
```
20层ABF GB200量产·28层Rubin批量·M10样品⚠️单源待核(2026-05-26互动易)·...
```

**改后**（两种格式选一）：
```
20层ABF GB200量产·28层Rubin批量·M10样品✅已双源核实(招商证券·2026-XX-XX)·...
```

或更详细版（保留来源 URL 备注）：
```
20层ABF GB200量产·28层Rubin批量·M10样品⚠️单源→✅(招商证券·2026-XX-XX + 中信证券·2026-XX-XX)·...
```

**修改流程**：
1. CC 输出 diff 给用户确认
2. 用户显式说「通过」后才写文件
3. 写入后必须做 JS 语法粗检 + git diff --stat 复核
4. 必须在 commit message 里写明：核实来源 + 改前 ⚠️ 标注 + 改后 ✅ 标注

## 适用范围

- 所有产业链 `*.manual.js` 文件的所有字段
  - `stocks[code].trendNote` / `logic` / `position` / `dims6[*].reason`
  - `chokePoints[code].verification.logic` / `logic` / `tags`
  - 任何未来新增字段
- HBM / 光模块 / 半导体设备 / 商业航天 等未来新增产业链同样适用

## 不适用范围（明确豁免）

- `pcb.auto.js` / `<chain>.auto.js` 的 `flag` 字段（程序生成·由 `scripts/calc_percentile.py` 自动实装）
- `index.html` 渲染层的 UI 提示（业务提示+自动判断·非手动层数据）
- `changelog` 数组里的历史变更标注（变更日志）
- 注释里的 ⚠️（如 `// ⚠️ §6.2 硬红线` · 治理纪律类·非数据类）

## 与现有纪律的关系

| 现有纪律 | 本 Skill 关系 |
|---|---|
| CLAUDE.md §6.4 media 单源直接拒绝 | 一致（执行更具体的判断标准）|
| CLAUDE.md §6.10 三重验证 | 一致（验证 + 数据冲突检查）|
| CLAUDE.md §6.11 精确 stock 列表 prompt 模板 | 互补（本 Skill 是其下游执行步骤）|

## 错误案例（CLAUDE.md §6.9 历史教训）

- **002288 超华科技** 已退市，但被纳入 P1 流程，豆包返回 13 项事件含 7 项 2026 幻觉事件被采信为 down 判定依据 → 本 Skill 第三步的「模糊表述 / 来源不完整 / 训练数据不算源」三道防线就是为此事故设计

## 操作快捷指令

CC 内部触发本 Skill 的简化指令：

```
/verify-single-source
```

自动扫描所有 `data/*.manual.js`，列出所有 `⚠️单源待核` 条目，按上述四步流程逐条执行。
